# Supabase/Postgres setup plan

Це не ламає поточну гру. Зараз гра далі працює через локальний JSON-файл на Render, а цей документ готує нас до нормальної бази даних.

## Навіщо це потрібно

JSON підходить для першого тесту: перевірити Telegram, Stars, рейтинг, адмінку.

Postgres/Supabase потрібен, коли ми хочемо:

- зберігати багато гравців стабільно;
- бачити історію оплат, дій, Zen-сесій і артефактів;
- робити графіки та аналітику;
- не боятися, що дані загубляться при перезапуску сервера;
- будувати інвентар, ліги, NFT-логіку і складну лабораторію.

## Що вже підготовлено

- `database/supabase-schema.sql` — структура майбутньої бази.
- `/api/admin/export` — експорт поточних даних з адмінки в JSON.
- `.env.example` — додані змінні для майбутнього підключення бази.
- сервер вже вміє працювати з Supabase через REST API.
- якщо Supabase не налаштований або впаде, сервер лишається на JSON fallback.

## Простий шлях підключення

1. Створити проект у Supabase.
2. Відкрити SQL Editor.
3. Вставити весь код з `database/supabase-schema.sql`.
4. Натиснути Run.
5. В Supabase скопіювати:
   - Project URL;
   - Service role key.
6. В Render додати змінні:
   - `DATA_BACKEND=supabase`
   - `SUPABASE_URL=...`
   - `SUPABASE_SERVICE_ROLE_KEY=...`
7. Натиснути Manual Deploy або дочекатися автоматичного деплою.
8. Відкрити `/admin` і подивитися блок `Data Control`.

Якщо все добре, там буде:

- `Active storage`: Supabase tables;
- `Migration`: Ready;
- `Supabase env ready`.

Якщо буде помилка, гра не повинна впасти. Вона покаже fallback і продовжить працювати через JSON.

Якщо Supabase порожній, а на сервері ще є старий JSON-файл з гравцями, сервер спробує автоматично перенести цей JSON у Supabase при першому старті.

## Як перенести поточні дані

1. Зайти в адмінку: `/admin`.
2. Натиснути `Export JSON`.
3. Отримати файл/вкладку з поточними `players`, `orders`, `payments`, `events`.
4. На наступному кроці ми зробимо імпорт цього файлу в Supabase.

## Важливо про ключі

`SUPABASE_SERVICE_ROLE_KEY` не можна вставляти в frontend-код або показувати гравцям. Він має бути тільки в Render Environment Variables.

Цей ключ дає серверу право записувати гравців, платежі та події в базу.

## Що важливо не плутати

- GitHub зберігає код.
- Render запускає сервер.
- Supabase/Postgres зберігає дані.
- Telegram Bot API проводить запуск Mini App, webhook і Stars.

Тобто GitHub не є базою даних для гри. Він потрібен для коду, історії змін і деплою.
