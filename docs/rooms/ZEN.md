# Zen Room Passport

## Role

Zen is the core product. The game exists to lead users into meditation through curiosity, progression, visuals, and reward.

Zen is not decoration. It is the differentiator.

## Current Elements

- Zen room prototype
- Visual meditation surface
- Sound concept
- Zen energy concept
- DNA/particle attention concept
- Nano pot / gene module prototype

## Adopted Blueprint Direction

Zen should feel like:

- a calm breathing app
- a living procedural chamber
- a meaningful daily ritual
- a resource generator for Zen/Calm Energy
- a place where Lab artifacts become emotionally valuable

Core flow:

```text
open Zen
choose session
choose sound/artifact
start breathing
interact with soft attention points
complete session
receive Zen/Calm Energy
save stats
```

## MVP Session Types

Use simple readable durations first:

| Session | Duration | Role |
| --- | ---: | --- |
| Quick Calm | 45 sec or 1 min | first-session and daily quick ritual |
| Deep Focus | 2 min | stronger reward and focus |
| Long Resonance | 5 min | deeper session later |

If UI space is tight, show only the first 1-3 options and keep them clear.

## Attention Points

Each session can spawn 3 soft attention points.

Rules:

- calm, not stressful
- visible enough to understand they are tappable
- never block main breathing text
- missed points reduce focus score slightly, not fail the session

## Artifact Input

Zen should eventually accept Lab-created artifacts:

- calm fragments
- strain echoes
- rare orbs
- featured BioCard/artifact

Artifact affects:

- visual field
- Zen/Calm Energy gain
- next capsule stability
- rare meditation visual

## Sound Direction

Sound is important but must be optional and user-initiated.

MVP soundscapes:

- Ocean Breath
- Forest Pulse
- Deep Space

Settings must include mute/volume controls.

## Legal Boundary

Zen can be framed as:

- relaxation
- focus
- calm ritual
- wellness-inspired gameplay

Avoid:

- medical claims
- curing anxiety
- treating illness
- guaranteed mental health outcomes

## Stable Rules

- Zen is not just another room. It is the main value.
- Sound and visuals must be calm, not noisy.
- Rewards should encourage completion, not only clicking.
- Session completion must be server-recorded before rewards become important.
- Zen should connect to Lab artifacts and Farm stability.
- Heavy GPU visuals are not allowed in Zen MVP.
